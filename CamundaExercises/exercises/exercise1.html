Model Twitter QA in the Camunda Modeler

Goal

The goal of this lab is to create a process application from the Maven archetype and use the generated process.bpmn to model the Twitter QA process in Camunda Modeler.

Short description

  • Download the Camunda Modeler.
  • Create a process application from the Camunda Archetype camunda-archetype-ejb-war and name it twitter-qa.
  • Open the generated empty process from the resources in the Camunda Modeler and model the process from Lab 0.

Detailed steps

  1. Download the Camunda Modeler from the official download page or use the version provided by your instructor.
  2. Unpack the downloaded archive and start the Modeler. Register the .bpmn, .cmmn and .dmn file extension with the Camunda Modeler.
  3. Start Eclipse and create a new Camunda BPM project in your workspace using our Maven archetypes. (docs.camunda.org/manual/latest/user-guide/process-applications/maven-archetypes/) Navigate to File -> New -> Project… -> Maven -> Maven Project.
  4. Check if you have configured the Camunda Archetype Catalog docs.camunda.org/manual/latest/user-guide/process-applications/maven-archetypes/#detailed-instructions in your Eclipse IDE.
  5. On the second page of the wizard, type camunda into the filter field and select the archetype camunda-archetype-ejb-war. Name the project twitter-qa.
  6. Have a look at which files have been generated by the archetype to get an overview.
  7. Open the empty process model generated by the archetype in the folder ‘src/main/resources’ with the Camunda Modeler and model the Twitter QA process from Lab 0.
  8. Save your work, which will save the BPMN 2.0 XML to your disk as well. If you want, take a look at it using the XML tab.
