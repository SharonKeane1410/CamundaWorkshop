Deploy the Prototype on Camunda BPM

Goal

The goal of this lab is to make your Twitter QA process executable and run it on the Camunda BPM platform installed on your machine.

Short description

  • Fill all required properties in the property panel of the Camunda Modeler to make the process executable. Model the services as script task with a script just print something to the console. Set the candidate group to management and model the conditions on the sequence flows as approved or not approved.
  • Build the process application and deploy it to the wildfly server.
  • Try your process in the tasklist filling the generic forms.

Detailed steps

  1. Set the candidate group of the user task to management.
  2. Set the conditions of the outgoing sequence flows of the exclusive gateway to #{approved} and #{not approved}.
  3. Morph the automated tasks into script tasks, set their script language to JavaScript and their script to: print('Hello World!\n');
  4. Now deploy the process to Camunda BPM: Right-click on the Eclipse project and select Run As -> Maven build. Then add the goals clean wildfly:deploy and press Run.
  5. After the build is finished, you can start your process using Camunda Tasklist localhost:8080/camunda/app/tasklist/.
  6. Fill the generic forms and walk through your process.
