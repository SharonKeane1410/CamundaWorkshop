Run the First JUnit Test

Goal

The goal of this lab is, that you build your first JUnit test case for a BPMN 2.0 process with the help of camunda-bpm-assert.

Short description

  • Strip down the process to the happy path. Morph the user task to a manual task. Set the conditions to true and false.
  • Prepare your content assist to the camunda-bpm-assert library.
  • Fill the generated InMemory JUnit test and assert that a started process instance is ended right after the start.
  • Have a look at the Camunda BPM Assert Library User Guide to get more information about the test library.
  • Run the JUnit test to see the green bar. Check the Output of the scipts in the console.

Detailed steps

  1. Edit your BPMN model in the Modeler and strip down the process to the happy path. Morph the user task to a manual task. Set the conditions on the sequence flows to ${true} and ${false}.
  2. Two JUnit test cases have been created in src/test/java. Open and inspect the ‘InMemoryH2Test’.
  3. In Eclipse go to Window -> Preferences -> Java -> Editor -> Content Assist -> Favorites -> New Type... and add the following type: org.camunda.bpm.engine.test.assertions.ProcessEngineTests
  4. Change the happy path test method that deploys the process using the @Deyployment annotation and start a process instance using the Runtime Service, which is available via runtimeService()
  5. Add an assert to verify, that the process instance is ended: You can use this code: assertThat(processInstance).isEnded();
  6. Have a look at the Camunda BPM Assert Library User Guide to get more information about the test library.
  7. Run the JUnit test (right click on the method and select Run As -> JUnit Test). You should get a green bar. Can you see the ouput of your script task in the console?
