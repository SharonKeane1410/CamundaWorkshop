Complete the Test Coverage

Goal

In this lab you add the Junit methods to complete the test coverage. Use camunda-bpm-assert-shortcuts and use process instance modifications for this.

Short description

  • Add a new JUnit test method with the process deployed.
  • Start the process directly before the unit you want to test.
  • Assert that the unit has the expected result.
  • Inspect the test coverage.

Detailed steps

  1. Add an new method to your JUnit test. Use the @Deployment annotation to deploy the process to test into the process engine.
  2. Create the process instance with the process instance modifications.
  3. Set the state of the process by setting all needed process variables with the right value.
  4. Move the token before the element you want to test.
  5. Assert the expected state of the process.
  6. Check the process test coverage in the target-folder.
