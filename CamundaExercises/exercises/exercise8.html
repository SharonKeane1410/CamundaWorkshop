Camunda BPM REST API

Goal

In this lab you get to know the REST API provided by Camunda BPM. The list of all API calls is in the docs.

Short description

  • Open a rest client and get all process definitions.
  • Start a process instance of the Twitter-QA process.
  • Query for tasks.
  • Claim and complete the tasks.
  • Check the result on Twitter.
  • Check the completed process instance.

Detailed steps

  1. Open a REST client like Postman: www.getpostman.com/ for Chrome or RESTClient for Firefox: addons.mozilla.org/de/firefox/addon/restclient
  2. The entry point for the REST API is: localhost:8080/engine-rest/
  3. Query for all process definitions with a get request to ensure that the Twitter QA Process is available. Get List
  4. Start a process instance of the latest version and pass the content variable. Start Instance
  5. Query for the user task of the started process instance. Get List
  6. Claim the task for the user demo. Claim
  7. Complete the task and pass the review result as a variable. Complete
  8. Check the completed process instance in the history. Get List
